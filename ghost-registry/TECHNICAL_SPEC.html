Ghost Registry (Ghosty)

Implementation Details for the Decentralized Web.

1. Introduction

Ghosty leverages the Freenet protocol to create a global, shared state of software packages. Logic is decentralised via WebAssembly (WASM) contracts, ensuring that updates are verifiable and unstoppable.

2. Data Models

The registry state is stored in a structured RegistryState map:

struct PackageEntry {
    owner_pubkey: Vec<u8>,
    versions: HashMap<String, String>, // Version -> CID
    latest_version: String,
    metadata: PackageMetadata,
}

3. Local Proxy API

The local daemon (ghostyd) intercepts requests from package managers:

  • GET /:package: Resolves latest version hash.
  • PUT /:package: Initiates a signed update to the P2P network.

4. Conflict Resolution

Using Commutative Monoids, Ghosty resolves simultaneous updates by prioritizing higher SemVer strings (e.g., 1.2.0 wins over 1.1.9), ensuring a consistent global "latest" pointer.
